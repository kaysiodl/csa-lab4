# Lisp - подобный язык

Основан на S-выражениях: весь код представляется в виде списков. Каждое выражение записывается в круглых скобках, где первый элемент - оператор или имя функции, а остальные - его аргументы.
Ниже приведена формальная грамматика языка в нотации Бэкуса-Наура, задающая допустимые конструкции программ.
```
<program> ::= <expr_list>

<expr_list> ::= <expr> | <expr> <expr_list>

<expr> ::= <number>
         | <string>
         | <identifier>
         | "(" <operator> <expr_list> ")"
         | "(" <comparison_operator> <expr> <expr> ")"
         | "(" "var" <identifier> <expr> ")"
         | "(" "set" <identifier> <expr> ")"
         | "(" "if" <expr> <expr> <expr> ")"
         | "(" "while" <expr> <expr_list> ")"
         | "(" "def" <identifier> "(" <param_list_opt> ")" <expr_list> ")"
         | "(" "call" <identifier> <arg_list_opt> ")"
         | "(" "print" <expr> ")"
         | "(" "read" ")"
         | "(" "read_str" ")"
         | "(" "len" <expr> ")"

<operator> ::= "+" | "-" | "*" | "/"

<comparison_operator> ::= ">" | "<" | "=" | "!="

<param_list_opt> ::= <param_list> | ε
<param_list> ::= <identifier> | <identifier> <param_list>

<arg_list_opt> ::= <arg_list> | ε
<arg_list> ::= <expr> | <expr> <arg_list>

<identifier> ::= <letter> <identifier_tail_opt>
<identifier_tail_opt> ::= <identifier_tail> | ε
<identifier_tail> ::= <letter> | <digit> | <identifier_tail>

<string> ::= "\"" <string_content_opt> "\""
<string_content_opt> ::= <string_content> | ε
<string_content> ::= <character> | <character> <string_content>

<number> ::= <digit> | <digit> <number>
<character> ::= <letter> | <digit> | " " | "," | "!" | "?"
<letter> ::= "a" | "b" | ... | "z" | "A" | "B" | ... | "Z"
<digit> ::= "0" | "1" | ... | "9"
```

## Семантика

- Все выражения вычисляются по стратегии "applicative order" - сначала аргументы, затем оператор.
- Все выражения возвращают значения.
- Любое выражение или вызов функции всегда возвращают последнее вычисленное выражение.
- Выполнение программы начинается с первого выражения, не считая объявления функций.
- Переменные создаются через `var`, область видимости — глобальная и функциональная. Из функции невозможно обратиться к переменным извне, следует передавать их как аргументы.
- Имена переменных и функций чувствительны к регистру.
- `set` - присвоить переменной значение.
- `if` вычисляет условие, затем один из блоков `expr_list`.
- `while` повторяет выполнение `expr_list`, пока условие истинно.
- `def` создаёт именованную функцию с параметрами и телом, поддерживается рекурсия.
- `call` вызывает уже созданную функцию.
- `read` - ввод числа в переменную.
- `read_str` - ввод строки в переменную, окончанием строки ожидается `\n`.
- Выражения арифметики (`+`, `-`, `*`, `/`) выполняются над числовыми значениями.

# Память
## Организация памяти
Модель памяти процессора:
- Архитектура Фон-Неймана (команды и данные в единой памяти).
- Память команд. Машинное слово -- 5 бит(команда) и 27 бит аргумент.
- Память данных. Машинное слово -- 32 бита, знаковое. 
- Адресация - абсолютная.


# Система Команд

## Формат инструкции
Инструкция имеет фиксированную длину **32 бита** (4 байта).
* Биты 31...27 - Код операции, 5 бит.
* Биты 26...0 -  Блок адреса, 27 бит. Используется для абсолютных адресов памяти, переходов или передачи констант. 27 бит позволяют адресовать до $2^{27}$ слов
*  Доступ к памяти осуществляется по адресу, хранящемуся в регистре PC. Изменить данный регистр можно следующими способами:
   - Инкрементировать данный регистр
    - Записи аргумента из машинного слова (при условных и безусловных переходах (jmp, jz, jn и тд))
* В командах `IN`/`OUT` используются младшие биты адресного поля для выбора порта (0–15).
  
| Название | Синтаксис | Описание                                               | Опкод (Bin) |
| :--- | :--- |:-------------------------------------------------------| :--- |
| **HALT** | `HALT` | Остановка выполнения программы                         | `00000` |

| **ADD** | `ADD` | Сложение первых двух элементов стека                   | `00100` |

| **MUL** | `MUL` | Умножение первых двух элементов стека                  | `00110` |
| **DIV** | `DIV` | Целочисленное деление второго элемента стека на первый | `00111` |
| **AND** | `AND` | Побитовое "И"                                          | `01000` |
| **OR** | `OR` | Побитовое "ИЛИ"                                        | `01001` |
